                              HOW IT WORKS (USER FLOW)
════════════════════════════════════════════════════════════════════════════════

SCENARIO 1: Opening PDF from WhatsApp
─────────────────────────────────────────────────────────────────────────────

User Action:
  1. Receives PDF in WhatsApp
  2. Long-presses the PDF
  3. Selects "Open with"
  4. Chooses "LearnDocs Hub"

System Process:
  1. Manifest triggers file handler
  2. file-handler.js receives the file
  3. File is converted to Blob
  4. URL parameters generated
  5. Redirects to /pdf-editor.html
  6. PDF auto-opens in viewer
  7. User can edit immediately

Result:
  ✓ PDF opens in 2-3 seconds
  ✓ Ready to edit
  ✓ All tools available
  ✓ Auto-save in 30 seconds


SCENARIO 2: Opening PDF from Email
─────────────────────────────────────────────────────────────────────────────

User Action:
  1. Opens email with PDF attachment
  2. Taps the attachment
  3. Selects "Open with" → "LearnDocs Hub"

System Process:
  1. Email app triggers file handler
  2. FileHandler intercepts attachment
  3. Converts to Blob format
  4. Navigates to PDF editor
  5. PDF loads automatically
  6. All editing tools available

Result:
  ✓ Fast opening
  ✓ Full editing capability
  ✓ Auto-save enabled
  ✓ Can export after editing


SCENARIO 3: Drag & Drop Upload
─────────────────────────────────────────────────────────────────────────────

User Action:
  1. Opens LearnDocs Hub
  2. Drags PDF from file manager
  3. Drops on browser window

System Process:
  1. JavaScript detects drop event
  2. File is extracted from drag data
  3. FileReader converts to Blob
  4. Auto-redirects to editor
  5. PDF opens immediately

Result:
  ✓ Instant opening
  ✓ No dialogs needed
  ✓ Seamless experience


════════════════════════════════════════════════════════════════════════════════
                              DRAFT SYSTEM DETAILS
════════════════════════════════════════════════════════════════════════════════

AUTO-SAVE MECHANISM:
  Trigger: Every 30 seconds during editing
  Action: Serializes PDF content + annotations
  Storage: Browser localStorage (10MB max)
  Silent: No interruption to user work
  Persistence: Survives browser restart

MANUAL SAVE:
  Trigger: User clicks "Save Draft" button
  Action: Immediate serialization
  Feedback: Success message appears
  Data: Includes timestamp and document name
  Retrieval: Can be loaded anytime

DRAFT DATA STRUCTURE:
  {
    id: 1714424357000,
    name: "Invoice_April.pdf",
    content: [ArrayBuffer with PDF bytes],
    annotations: [
      { type: "text", x: 100, y: 150, text: "Paid", color: "#000000" },
      { type: "rect", x: 50, y: 50, width: 200, height: 100 },
      ...
    ],
    savedAt: "2026-04-29T10:32:37Z",
    updatedAt: "2026-04-29T10:35:41Z"
  }

STORAGE LOCATION:
  Browser: localStorage['pdf_drafts']
  Scope: Per domain (learn-doshub.vercel.app)
  Access: Only from same domain
  Lifetime: Until user clears browser data
  Backup: None (local only)

                              TECHNICAL ARCHITECTURE
════════════════════════════════════════════════════════════════════════════════

BROWSER APIS USED:
─────────────────────────────────────────────────────────────────────────────

1. File Handler API
   - Intercepts file opens from system
   - Routes .pdf files to /pdf-editor.html
   - Available on: Android, iOS, Desktop

2. Web Share API
   - Share edited PDFs
   - Works with system share sheets
   - Fallback to clipboard links

3. Drag & Drop API
   - Drop zone detection
   - File extraction from drag event
   - MIME type validation

4. FileReader API
   - Convert files to Blob
   - Generate data URLs
   - Binary file handling

5. LocalStorage API
   - Persist drafts locally
   - Key-value storage
   - 10MB per domain limit

6. Service Worker API
   - Cache app files
   - Offline support
   - Background sync (future)

                            SUPPORT & TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

COMMON ISSUES:
─────────────────────────────────────────────────────────────────────────────

Issue: PDF doesn't open from WhatsApp
Solution:
  1. Check if manifest.json is valid
  2. Verify HTTPS is enabled
  3. Clear browser cache
  4. Reinstall PWA
  5. Check console for errors

Issue: Drafts not saving
Solution:
  1. Check if localStorage is enabled
  2. Verify quota not exceeded
  3. Check browser privacy settings
  4. Try manual save
  5. Check browser console

Issue: File too large to open
Solution:
  1. Split PDF into parts
  2. Compress PDF first
  3. Use smaller PDFs for testing
  4. Check file size limit


DEBUGGING:
  - Open browser console (F12)
  - Look for [v0] tagged messages
  - Check for any JavaScript errors
  - Verify file-handler.js is loaded
  - Check manifest.json is valid
